Key Conecpts & Defintion

This section helps define and explains the key concepts that play important roles in understanding more about web servers and their way of operating.

Untangling the Web

Web Server
A computer system that delivers web content to end users via the internet using the HTTP protocol.

Web servers are software that allow users and website to communicate through gathered info,and recieve request using HTTP or HTTPS.


HTTP
HyperText Transfer Protocol: The set of rules used to transfer data and allows you to request and recieve info over the internet.

HTTP plays a major role in the call and response of request on the web.It is similar to HTTPS and works in hand with web servers.


HTTPS
HyperText Transfer Protocol Secure: The set of rules used for transferring data over servers and broswers securely.

HTTPS is the upgraded,encrypted version of HTTP.HTTPS uses scrambled or "encrypted" data which makes it hard for the average person to understand what's being dispalyed.


Latency
Latency is the delayed time between a input and it's response to carrying out it's instructions.

Latency can be both good or bad depending on how fast or slow the response time may be, but that would depend on certain variables affecting it.


World Wide Web
The World Wide Web also known as www at the start of most websites is the gateway to viewing info through hyperlinks.

The World Wide Web is what directs your browser to a server across the internet related to the url searched after.


IP Address
An Internet Protocol address is a special and unique set of numbers that label a device that's connected to a network .

IP addresses are almost like a home address but for devices on networks.They allow for easily being able to locate and route between devices.


Web Browsers
Web Browsers are software applications that retrieve,process,and display web pages with use of the internetIt's a primary software used to navigate through the World Wide Web.

Web Browsers are what gather code like HTML and CSS and make it readable to devices that load the link.
